# Single-Cycle Parallel Hardware Accelerator for Conway's Game of Life

A high-throughput, spatially parallelized 2D cellular automaton accelerator implemented in SystemVerilog. This design eschews traditional sequential software-style loops in favor of a fully unrolled hardware architecture, computing state updates for an entire 16x16 universe simultaneously within a single clock cycle.

---

## Architectural Overview

Most software implementations of Conway's Game of Life rely on nested iterations that process cell logic sequentially (O(N^2) time complexity). This project implements a hardware-native spatial architecture that scales the throughput to a constant O(1) per generation.

### Spatial Compute Fabric
The design instantiates 256 distinct Arithmetic Logic Units (ALUs) operating completely in parallel. On every rising edge of the clock, each cell independently evaluates its next-state logic based on its immediate 8-neighbor neighborhood.

### Toroidal Routing Fabric (grid_router.sv)
To completely eliminate control-flow overhead and boundary-checking stalls, a dedicated routing module wraps the edges of the 16x16 grid into a continuous toroidal (donut-shaped) space. 

> **Silicon Optimization Note:** The grid_router is implemented as pure combinational assignment logic using the Verilog concatenation operator {}. It synthesizes down to zero physical logic gates and zero propagation delay, acting purely as a static copper wiring fabric that pre-aligns every cell's neighbors.

---

## Algorithmic Mapping to Silicon

The behavior of each individual cell ALU is mapped directly from a 3-variable Karnaugh Map evaluating the Left (L), Center (C), and Right (R) behavior of the shifted grids. The logic minimizes down to a highly optimized Sum-of-Products form:

$$\text{Next State} = (C \oplus R) + (C \cdot \overline{L})$$

This minimal boolean footprint allows the entire 256-cell matrix to synthesize into highly compact Look-Up Tables (LUTs) and flip-flops, minimizing area utilization while maximizing maximum clock frequency (Fmax).

---

The design was verified in AMD Xilinx Vivado using a self-contained SystemVerilog testbench (tb.sv). The simulation initializes the universe with a classic 5-cell Glider pattern to test both inner-grid state transitions and wrapped boundary conditions.
